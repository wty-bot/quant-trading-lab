## 2. 主策略必须修订的内容

### 2.1 价值模块去冗余

当前问题：

- `book_to_market_ratio_ttm` 与 `pb_ratio_ttm` 本质重复。
- 同时纳入会导致 PB / B/P 信息重复计权。

修订方案：

- 保留 `book_to_market_ratio_ttm`。
- 去掉 `pb_ratio_ttm` 在价值模块中的直接使用。
- `pe_ratio_ttm` 负值或异常值置空，不参与价值分。
- 价值模块暂定为：

```text
value = mean(z(book_to_market_ratio_ttm), -z(pe_ratio_ttm))
```

若后续确认有可靠的 `earnings_yield` 或 `sales_to_price` 缓存，再作为增强版本加入，不作为 baseline 前提。

### 2.2 动量默认改为 12-1 月

当前问题：

- 120 日动量混入短期反转，经济含义不够清晰。

修订方案：

- baseline 默认使用 12-1 月动量：

```text
momentum_12_1 = close[t-21] / close[t-252] - 1
```

- 120 日动量只作为敏感性测试，不作为主策略默认。

### 2.3 质量模块升格现金流质量

当前问题：

- ROE、ROA、毛利率均偏盈利能力，同质性较高。
- A 股存在盈余管理、非经常性损益和保壳动机。

修订方案：

- `ocf_to_debt_ttm` 升为核心质量因子。
- baseline 质量模块：

```text
quality = mean(
  z(return_on_equity_ttm),
  z(return_on_asset_ttm),
  z(gross_profit_margin_ttm),
  z(ocf_to_debt_ttm)
)
```

- 报告中承认：质量因子基于财报口径，未完整剔除非经常性损益。

### 2.4 金融行业处理

当前问题：

- 银行、非银金融、房地产的会计口径与一般企业不同。
- 毛利率、OCF/Debt、ROE、PB 对金融股含义不同。

baseline 选择：

- 剔除金融行业。
- 建议剔除：
  - 银行
  - 非银金融
- 房地产是否剔除视行业字段可用性决定；若易于识别，也剔除。

报告说明：

- 本策略定位为非金融企业多因子选股系统。
- 金融行业需要单独因子体系，不纳入 baseline。

### 2.5 上市时间过滤改为 252 个交易日

当前问题：

- 180 自然日过滤偏短，A 股 IPO 效应更长。

修订方案：

- 改为至少 252 个交易日价格历史。
- 同时满足动量、波动率计算所需历史长度。

### 2.6 停牌 / 复牌规则精确化

当前问题：

- “可交易时卖出”表述过粗。

修订方案：

- 调仓日若持仓股票停牌：
  - 不发出卖出委托；
  - 保持原持仓；
  - 继续按停牌价计入组合估值，遵循 RQAlpha Plus 撮合/估值规则。
- 股票复牌后：
  - 在下一个可交易调仓检查点重新排名；
  - 若不满足持仓条件或跌出卖出阈值，则卖出。

### 2.7 调仓频率敏感性修订

当前问题：

- 当前因子缓存为月度快照，双周调仓会引入因子前值填充或插值问题。

修订方案：

- baseline：月度调仓。
- 敏感性测试：月度 vs 季度。
- 删除“双周调仓”作为默认敏感性测试。
- 若未来加入双周，必须明确使用最近月度因子前值填充。

## 3. baseline 策略设置

baseline 版本用于先跑通端到端回测，不追求一次性解决所有增强项。

### 3.1 股票池

每个调仓日从历史 A 股基础信息快照出发：

1. `type == CS`
2. `status == Active`
3. 非 ST / *ST
4. 非停牌
5. 剔除金融行业
6. 上市满 252 个交易日
7. 收盘价 `close > 2`
8. 过去 20 个交易日平均成交额 `> 20,000,000`
9. 市值 `market_cap > 2,000,000,000`

### 3.2 因子标准化

baseline 先采用全市场 winsorize + z-score，报告中补充行业/市值暴露分析。

后续增强版本再做行业内标准化。若做行业内标准化，规则为：

- 行业样本数 >= 30：行业内 winsorize + z-score。
- 行业样本数 < 30：回退全市场 winsorize + z-score，或合并为“其他”。

### 3.3 综合评分

baseline 保留原四模块框架，但修正因子构成：

```text
score = 0.35 * quality
      + 0.25 * value
      + 0.25 * momentum
      + 0.15 * low_risk
```

模块定义：

```text
quality = mean(ROE, ROA, GrossMargin, OCF_to_Debt)
value = mean(B/P, -PE)
momentum = 12-1 month momentum
low_risk = -120 trading day realized volatility
```

权重说明：

- 权重为经济直觉驱动，不是优化器拟合结果。
- 报告必须展示四个模块得分相关矩阵，解释信息重叠。

### 3.4 持仓与交易

- 买入：综合分前 30 名。
- 卖出缓冲：跌出前 60 名卖出。
- 权重：baseline 等权，目标总仓位 98%，保留 2% 现金。
- 不使用杠杆。
- 不做空。
- 不做择时。
- 不设个股止损。

合理性说明：

- 等权用于降低过拟合和解释成本。
- 不做择时是为了检验横截面选股能力。
- 不做止损是为了保持低频基本面/动量策略的一致性，避免短期噪声和止损参数过拟合。

## 4. 必须补充的分析与报告内容

### 4.1 PIT 与前瞻偏误说明

报告必须写清楚：

- 因子缓存是在 2026-06-16 通过逐调仓日调用生成。
- 每期使用 `date = rebalance_date` 的因子截面。
- 回测时只读取本地静态缓存，不动态调用 RQData。
- 依赖米筐 RQData 因子接口的日期口径；不手工使用报告期数据向前填充。

当前已有证据：

- `factor_monthly_full_history_summary.json` 中有 259 个逐调仓日 batch 记录。
- `factor_monthly_full_history.pkl` 中记录了 259 个调仓日和 `failed_dates = []`。

### 4.2 幸存者偏差说明

报告必须写清楚：

- 股票池来自历史基础信息月度快照与历史价格文件。
- 不是用当前存续股票回填历史。
- 已退市或历史存在的股票只要当期在数据中存在，即参与当期候选池。

当前已有证据：

- `all_instruments_monthly_full_history.pkl`
- `stocks.h5`
- `stock_local_coverage.csv`

### 4.3 隐性市值暴露

必须做：

- 子因子得分与 `ln(market_cap)` 的截面相关或回归系数时间序列。
- 组合持仓市值暴露 vs 基准市值暴露。

目的：

- 检查质量、价值、低波模块是否共同形成大市值或小市值押注。
- 避免把规模因子误解释为 Alpha。

### 4.4 行业暴露与归因

必须做：

- 组合行业权重时间序列。
- 组合行业权重 vs 基准行业权重。
- 至少报告若干关键年份的行业超配/低配。

若时间允许：

- 做简化 Brinson 归因。

### 4.5 模块相关矩阵

必须做：

- quality / value / momentum / low_risk 四个模块得分的历史平均截面相关矩阵。
- 报告说明模块之间存在信息重叠，权重不是四个完全独立来源。

### 4.6 换手率与交易成本

必须报告：

- 单期换手率。
- 年化换手率。
- 交易成本后净值。
- 交易成本对年化收益的影响。

### 4.7 A 股特异性风险说明

报告局限性中必须包含：

- 壳价值可能污染低估值因子。
- 盈余管理和非经常性损益可能污染质量因子。
- 月末调仓可能受到月末效应、季末做账、长假前流动性异常影响。
- 低波动因子可能引入大市值偏好。
- 本策略不做止损、不做择时、不做深度容量建模。

## 7. 暂不纳入 baseline 的内容

以下内容不进入第一版 baseline，但可作为后续增强：

- 完整 Brinson 归因。
- beta / idiosyncratic volatility 替代低波动。
- 个股止损。
- 市场择时或现金仓位模型。
- 分层调仓：质量季度、价值月度、动量双周。
- 金融行业独立因子模型。
- 机构级容量分析。

这些内容在课程项目中属于增强项，不应阻塞第一次端到端回测。
